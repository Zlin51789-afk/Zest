# 让全世界通过 chipgo.net 访问你这台 Mac（不用 Vercel）

你要的效果：

- 域名 **chipgo.net** 在公网打开 → 请求到你 **家里/公司的这台电脑**
- PDF、账号在 **`~/chipgo-private`**，不上传 GitHub
- **电脑要长期开机**，程序要一直运行

这和「只改本机 hosts」完全不同：必须让 **互联网能连到你电脑的公网入口**。

---

## 推荐方案：Cloudflare Tunnel（不用路由器端口映射）

适合家庭宽带、没有固定公网 IP。流量：用户 → Cloudflare → 隧道 → 你电脑 `localhost:80`。

### 1. 安装 cloudflared（Mac）

```bash
brew install cloudflared
```

### 2. 登录并创建隧道

```bash
cloudflared tunnel login
cloudflared tunnel create chipgo-home
```

记下输出的 **Tunnel ID**。

### 3. 配置文件

复制项目里的示例：

```bash
cp config/cloudflared.yml.example ~/.cloudflared/config.yml
```

编辑 `~/.cloudflared/config.yml`，把 `<TUNNEL_ID>` 换成你的 ID；把 `credentials-file` 路径改成：

`~/.cloudflared/<TUNNEL_ID>.json`

### 4. 把域名指到隧道（在 Cloudflare 管理 DNS）

若 **chipgo.net 已在 Cloudflare**（域名 NS 改到 Cloudflare）：

```bash
cloudflared tunnel route dns chipgo-home chipgo.net
cloudflared tunnel route dns chipgo-home www.chipgo.net
```

若域名还在 **聚名网 / Vercel DNS**：需要先把域名的 **NS（域名服务器）** 改为 Cloudflare 提供的 NS，或在聚名网添加 CNAME（按 Cloudflare 面板提示）。  
**必须停用** 原来指向 Vercel 的解析（如 `76.76.21.21` 或 `ns*.vercel-dns.com`），否则公网仍会打开 Vercel 而不是你电脑。

### 5. 在本机启动网站 + 隧道（两个终端）

**终端 A**（网站，数据在 `~/chipgo-private`）：

```bash
cd /Users/a1111/Documents/cursor/web2
npm run setup:local    # 首次
npm run public
```

**终端 B**（隧道）：

```bash
cloudflared tunnel run chipgo-home
```

浏览器访问：**https://chipgo.net**（Cloudflare 自动 HTTPS）。

在 `.env` 增加（隧道 HTTPS 登录 Cookie 需要）：

```bash
PUBLIC_HTTPS=1
```

## 方案 B：路由器端口转发 + 聚名网 A 记录

适合有 **固定公网 IP** 或会用 **动态 DNS（DDNS）**。

### 1. 查公网 IP

```bash
curl -s ifconfig.me
```

### 2. 路由器里设置

- 外网 **80**（和可选 **443**）→ 转发到你 Mac 的局域网 IP（如 `192.168.1.9`）的 **80** 端口
- Mac 防火墙允许入站 80

### 3. 聚名网改 DNS（停用 Vercel）

登录聚名网 → 域名 **chipgo.net** → DNS 解析：

| 记录 | 类型 | 值 |
|------|------|-----|
| @ | A | 你的公网 IP |
| www | A 或 CNAME | 同上或 `chipgo.net` |

**删除或停用** Vercel 的 NS（`ns1.vercel-dns.com` 等）和指向 `76.76.21.21` 的记录。

### 4. 启动本机服务

```bash
cd /Users/a1111/Documents/cursor/web2
npm run public
```

公网访问：**http://chipgo.net**（建议再加 HTTPS，可用 Cloudflare 仅做 CDN/证书）。

## 必须知道的风险与条件

1. **电脑关机 = 网站全站不可用**（没有 Vercel 备份）。
2. **家庭上行带宽** 决定多人同时下载 PDF 的速度。
3. **安全**：公网暴露后务必改强密码、`AUTH_ADMIN_SECRET`、考虑仅允许 VPN/内网访问。
4. **AI**：本机需配置 OpenClaw 或 `MOONSHOT_API_KEY`（密钥只在 `.env`，不上 Git）。
5. 从 Vercel 迁走后，在 Vercel 控制台可 **暂停/删除** 该项目，避免混淆。

| 命令 | 作用 |
|------|------|
| `npm run setup:local` | 数据放到 `~/chipgo-private` |
| `npm run public` | 本机 80 端口对外服务（局域网+公网转发目标） |
| `npm run local` | 仅本机 hosts 测试（**不能**让全世界访问） |
| `cloudflared tunnel run chipgo-home` | 公网隧道（配合 `npm run public`） |

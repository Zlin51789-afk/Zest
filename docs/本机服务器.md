# 用本机当服务器（隐私文件不上 GitHub）

线上 **chipgo.net（Vercel）** 会把仓库里的 `data/**` 打包部署，**不适合放隐私 PDF**。  
本机模式：程序跑在你电脑上，文档、账号只读**本机磁盘**，无需上传到 GitHub。

## 1. 准备隐私文件目录（推荐）

在电脑任意位置建文件夹，例如：

```text
~/chipgo-private/
  documents/     ← 所有可下载 PDF、手册（结构同 data/documents）
  knowledge/
    faq.json
  progress/
    project-alpha.json
  accounts.local.json   ← 登录账号（复制 example 后改名）
```

把 PDF 放进 `documents/`，不要放进 Git 仓库。

## 2. 配置 `.env`（项目根目录）

复制 `.env.example` 为 `.env`，并增加本机路径（按你的实际路径修改）：

```bash
# 本机一体服务端口
PORT=3001
HOST=0.0.0.0

# 隐私数据目录（绝对路径）
DOCUMENTS_DIR=/Users/你的用户名/chipgo-private/documents
KNOWLEDGE_PATH=/Users/你的用户名/chipgo-private/knowledge/faq.json
PROGRESS_DIR=/Users/你的用户名/chipgo-private/progress
ACCOUNTS_FILE=/Users/你的用户名/chipgo-private/accounts.local.json

# AI：二选一
# ① 本机 OpenClaw（推荐，密钥不出本机）
AI_BACKEND=openclaw
OPENCLAW_GATEWAY_TOKEN=你的token

# ② 或直连 Moonshot（密钥在 .env，仍不会上传 GitHub）
# AI_BACKEND=moonshot
# MOONSHOT_API_KEY=sk-...
```

`accounts.local.json` 格式见 `data/accounts.local.json.example`。

## 3. 启动

```bash
cd /path/to/web2
npm install
npm run serve:local
```

浏览器打开：**http://localhost:3001**

同一 WiFi 下其他设备可用：**http://<你电脑的局域网 IP>:3001**（在系统设置里查看 IP）。

## 4. 与 GitHub / Vercel 的关系

| 项目 | 本机服务器 | chipgo.net（Vercel） |
|------|------------|----------------------|
| 隐私 PDF | 只在你电脑磁盘 | 仓库里有的才会上线 |
| 关机后 | **不能访问**（除非再做内网穿透） | 仍可访问 |
| 改账号/文档 | 改本地文件，重启 `serve:local` | 需 commit + 部署 |

**不要**把 `.env`、`accounts.local.json`、隐私 PDF 提交到 Git。`.gitignore` 已忽略 `data/documents/**` 与 `data/accounts.local.json`（若历史上已提交过 PDF，需用 `git rm --cached` 从仓库移除，文件仍保留在本机）。

## 5. 开发模式（可选）

前后端热更新：

```bash
npm run dev
```

访问 http://localhost:5173（API 代理到 3001）。同样会读取 `.env` 里的 `DOCUMENTS_DIR` 等变量（Node 20+ 可在启动命令加 `--env-file=.env`）。

## 6. 已从 Git 跟踪的文档

若仓库里曾经提交过 `data/documents` 下的 PDF，它们仍会留在 GitHub 历史中。要从**今后**的提交里去掉（文件仍留在本机）：

```bash
git rm -r --cached data/documents/
git commit -m "Stop tracking document binaries; use local DOCUMENTS_DIR"
```

之后只在 `DOCUMENTS_DIR` 指向的文件夹里维护隐私文件。

# 所有文件都在你这台电脑上

**日常约定（2026-05 起）**：不再使用 GitHub / Vercel 更新站点与文档。新文件放进 `~/chipgo-private/documents/`，需要纳入检索时告诉助手即可，**无需** commit 或部署。

公网访问：**https://chipgo.net** → 本机 Mac（需保持 `npm run public` 或网站 + `cloudflared tunnel run chipgo-home` 运行）。

## 两种用法

| 目标 | 做法 |
|------|------|
| **只有你自己电脑** 用 chipgo.net 打开 | `npm run local`（改 hosts，别人访问不到） |
| **全世界** 都用 chipgo.net 访问你这台 Mac | 看 **[公网部署-本机当服务器.md](公网部署-本机当服务器.md)**，用 `npm run public` + 隧道或路由器 |

## 仅本机测试（改 hosts）

本机把 **chipgo.net** 指到 127.0.0.1（**只对你这台 Mac 生效**）。

### 启动

```bash
cd /Users/a1111/Documents/cursor/web2
npm run local
```

按提示输入 **Mac 密码**（用于写入 hosts + 使用 80 端口）。

浏览器打开：**http://chipgo.net**（不要加 `:3001`）

### 数据目录

```text
/Users/a1111/chipgo-private/
```

### 仅配置 hosts、暂不启动

```bash
npm run hosts:chipgo
```

### 仍想用 localhost:3001

```bash
npm run local:3001
```

## 说明

| 问题 | 答案 |
|------|------|
| 新 PDF 放哪？ | `~/chipgo-private/documents/`（可分子文件夹，如 `JFM9系列FPGA/`） |
| 别人还能打开 chipgo.net 吗？ | 已改 Cloudflare NS 后，公网走本机隧道；Mac 关机则全站不可用 |
| 关掉本机服务后 | 浏览器访问 chipgo.net 会连不上 |
| 恢复不用本机域名 | 编辑 `/etc/hosts`，删掉含 `# chipgo-local` 的那一行 |
